Parent host (raw postMessage — no nbridge)

This outer page implements the host side of the nbridge iframe contract by hand: it listens for message events from the embedded child, checks origin and source, acks the handshake, answers parent:getTitle requests, and can push messages into the child with iframe.contentWindow.postMessage. The child (inside the dashed border) uses nbridge and auto-detects the iframe platform.

<iframe src="/child.html"> — child app using nbridge

Host contract implemented here

  • Accept only events where event.source === iframe.contentWindow and event.origin === location.origin
  • __nbridge_handshake__ → reply __nbridge_handshake_ack__
  • parent:getTitle → reply parent:getTitle_response with the same id
  • __nbridge_batch__ → unpack payload.messages

Push into the child

Sends { type: "parent:ping" } into the iframe; the child's bridge.on("parent:ping") listener logs it.
